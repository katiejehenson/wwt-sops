## What's in This Repository

This repository contains internal SOPs and reference guides for the virtual admin (VA) supporting Weindling World Travel. All documents are HTML files that open directly in any browser — no software required.

## Documents

### 1. `weindling-tern-guide.html`
**Tern Workflow & Template Reference Guide**

The master reference for everything in the Tern CRM — workflows, email templates, forms, task lists, vendor commissions, and training videos.

**Tabs included:**
- **Summary** — Library counts at a glance, quick links (Calendly, Hotel Portal, Loom SOP)
- **SOP & Workflows** — Full step-by-step guide for Barbara and the VA; workflow trigger map; timed email reference
- **Travel Joy Templates** — All 55 email templates with category, type, and active status
- **Tern Workflows** — Hotel Only, Lead, and Pre/Post task lists and automations with Active/Inactive status
- **Forms** — All 14 client-facing forms
- **Task Lists** — Full Trip (19 tasks) and Hotel Only (6 tasks) with trigger types and offsets
- **Payment Reminders** — Setup steps and payment calculator guide
- **Vendor Commissions** — Invoice and payment instructions for each vendor
- **Training** — Video library with direct links (Axus, Airtable, NTCE Pay, Canva, Loom)

---

### 2. `WWT_Itinerary_SOP.html`
**Axus Itinerary Build & Review SOP** *(for VA use)*

Standard operating procedures for building final client itineraries in Axus and conducting a thorough pre-send quality review — based on Barbara's confirmed itinerary standards.

**Tabs included:**
- **Overview** — VA scope, two itinerary types (Proposal vs. Final), and the three-phase process
- **Part 1: Building** — Step-by-step entry for every element (flights, transfers, hotels, activities, dining, T&Cs, image credits)
- **Part 2: Images** — Barbara's image standards; what's acceptable vs. rejected; where to source images and in what priority order; image assignment by section type
- **Part 3: QA Review** — Six checklist categories: logical flow & timing gaps, missing information, supplier T&C accuracy, helpful information, proofreading, and proposal-specific checks
- **Element Standards** — Quick-reference tables for each element type showing required fields for Final vs. Proposal
- **Final & Proposal Diff** — Exactly what changes (pricing removed) vs. what stays (T&Cs, descriptions, images) when converting
- **Common Errors** — Flagged error types with real examples from reference itineraries

## How to Use These Files

1. Click the file name in GitHub to open the raw file, then click **Download** — or clone the repo locally.
2. Open the `.html` file in any web browser (Chrome, Safari, Firefox).
3. Use the tab navigation at the top of each document to jump between sections.
4. These are read-only reference documents. Do not edit them without Barbara's approval.

---

## Key Rules for the VA

- **Never send an itinerary to a client.** Always route back to Barbara for final review and send.
- **Never activate inactive automations in Tern** without Barbara's direction.
- **Never copy Travel Biz Boss cruise templates** into the library — access them from the shared folder only.
- **Never change trip statuses in Tern** without Barbara's approval.
- **Never edit master workflows** in Tern.
- **Always flag issues** in itineraries rather than fixing content unilaterally — send a QA note to Barbara via Tern.
