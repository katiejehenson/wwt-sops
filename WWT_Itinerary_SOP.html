How to Use This SOP

Barbara's standard: Every itinerary that leaves this agency must look polished, read clearly, and feel like it was built with care for that specific client. The VA's job is to build the itinerary structure faithfully, select images that are crisp and destination-appropriate, and conduct a thorough review before it ever reaches Barbara for a final check. Never send an itinerary to a client — always route back to Barbara.

Two Itinerary Types

Proposal — Sent before booking is confirmed. Includes pricing, room rates, confirmation references. Example: Greece / Pekin Family.

Final Itinerary — Sent once everything is booked and confirmed. Pricing is removed. Record locators, confirmation numbers, and all logistics are finalized. Example: Morocco / Chang Homer.

VA Scope of Work

  • Build itinerary structure in Axus per Barbara's instructions
  • Source and upload images for every section
  • Enter all supplier details, contact info, and descriptions
  • Conduct the full QA review checklist
  • Flag issues back to Barbara — never fix content unilaterally
  • Never send the itinerary to the client

01

Building the Itinerary in Axus

📋
Always start from Barbara's notes, the supplier confirmation documents, and any existing proposal. Build section by section in chronological order. Do not skip sections — placeholder content is not acceptable.
Itinerary Structure & Order
1

Header / Cover Page

Axus auto-populates the WWT logo and Barbara's contact info. Verify these are correct. Set the itinerary title to: [Client Last Name(s)] [Destination] [Trip Theme] — e.g., "Chang Homer Morocco Adventure" or "Pekin Family Greece Empty Nest Adventure." Set trip date range in the subtitle bar. Upload the hero cover image (see Image standards).

2

Pre Travel Section (always first)

This section always appears before Day 1 and contains four standard sub-sections in this order:

  1. IMPORTANT: Travel Policies — Use Barbara's standard template. Do not edit content.
  2. Before You Fly Reminders — Standard template. Confirm the Axus App download reminder is included.
  3. Emergency Contacts in [Destination] — Pull from supplier documents. Include the local contact names, WhatsApp group setup info, email, and phone. If the supplier uses a group chat system (as in Greece), copy that exactly.
  4. Preparing for Your Trip to [Destination] — Destination-specific section sourced from the supplier. Includes weather, dress code, currency, tipping guidelines, packing tips, etc. If the supplier provides this content, use it verbatim.

For trips with guide-specific information (like Morocco), add a Guide Information sub-section after the destination prep content.

3

Day-by-Day Sections

Each day gets its own date header (e.g., "Saturday · April 18th"). Within each day, add elements in chronological order. The standard element types are:

  • Flights — Always first if departing that day
  • Transfers / Arrivals — Including VIP services
  • Hotel / Accommodation — Immediately after arrival transfer
  • Activities / Tours / Experiences — In time order
  • Dining reservations — Evening reservations appear at the end of the day they occur
4

Supplier Terms & Conditions

T&Cs appear as the very last item in the itinerary — after the final day's content, including the return flight. In Axus, add each supplier's T&Cs as a document/note entry. The Morocco itinerary shows "Shin Luxury Tours Terms & Conditions" placed after the final flight. Include:

  • Supplier overview / mission statement
  • Vehicle and guide standards
  • Inclusions and Exclusions lists
  • Payment terms and cancellation policy
  • Any special service notes (e.g., Fast Track limitations)
5

Image Credits

If any images are sourced from Unsplash or a credited photographer, add the credit line in the footer or as a note on the cover image. Example from the Greece proposal: "Image credits: Spencer Davis on Unsplash." Barbara expects proper attribution when stock images are used.

Entering Each Element
6

Flights

Required fields for every flight:

  • Record locator (PNR)
  • Carrier name and flight number
  • Departure airport code + time
  • Arrival airport code + time (note +1 for next-day arrivals)
  • Flight duration
  • Passenger names and seat assignments (when available)
  • Baggage allowance if a specific fare class applies (e.g., ComfortFlex)

Full airport names appear in the section title (e.g., "New York, New York John F. Kennedy International Airport to Casablanca, Casablanca Mohammed V International Airport"). The airline name appears in italics below.

7

Transfers

Required fields:

  • Driver/company name and phone number
  • Departure location and arrival location
  • Vehicle type
  • Number of passengers
  • Duration
  • Rate (proposal only — remove for final)

For VIP airport services, list the full service description as provided by the supplier, including exactly what the VIP assistant will do (welcome at gate, fast-track, luggage, escort, etc.).

8

Hotels / Accommodation

Required fields:

  • Hotel name, phone number, full address
  • Check-in and check-out dates and times
  • Number of nights
  • Hotel description (from supplier or hotel website)
  • Amenities list (e.g., daily breakfast, welcome drink, spa credit)
  • Room Category, Bedding type, Room description
  • Confirmation reference number
  • Number of guests
  • Nightly room rate and total price (proposal only)
  • Payment notes if applicable
9

Activities / Tours / Experiences

Required fields:

  • Activity name
  • Supplier phone number (in the title area, linked)
  • Start time, End time, Duration
  • Meeting point
  • Description (from supplier documents)
  • Included items list
  • Notes section: practical tips, dietary notes, attire recommendations, difficulty level, etc.
  • Rate (proposal only)
✅
Any supplier notes that are written in orange/colored text in Barbara's reference itineraries are internal or client-facing advisories — keep these in the Notes field in Axus, styled appropriately so they stand out.
10

Dining Reservations

Required fields:

  • Restaurant name + "· Dining Reservation" appended to the title
  • Phone number
  • Full address
  • Website URL (when available)
  • Reservation time (shown as Start Time)
  • Confirmed By field (e.g., "Booked On Own" if client booked directly)

If dinner cost is not included in the package, add a note: (Cost of dinner is not included).

11

Restaurant Recommendations / Free Time Sections

When clients have unstructured time, add a section titled "Restaurant Recommendations and Suggestions of What to do on your own." Upload a relevant destination image. Link to the corresponding Axus document (e.g., "Athens Recommendations") so clients can access a curated list.

02

Image Standards

Barbara is particular about images. Every section must have a hero image. Images must be crisp, clear, high-resolution, and directly relevant to the specific place, hotel, restaurant, or experience — not generic stock photos. A blurry, pixelated, or off-topic image reflects poorly on the agency and must be replaced before the itinerary is reviewed.
Image Quality Standards
✓ Acceptable Images
  • High resolution — minimum 1200px wide for hero images
  • Crisp focus throughout — no motion blur, no soft focus on key subjects
  • Bright and well-lit — no underexposed or muddy images
  • Specific to the actual property, restaurant, or destination
  • Landscape / wide-format images for the hero banner slot
  • True to what the client will experience (not overly staged or AI-generated)
  • Professional photography from hotel/supplier websites
  • Credited Unsplash photos when supplier images are unavailable
✗ Do Not Use
  • Blurry or pixelated images — even slightly blurry is a rejection
  • Low-resolution images that look acceptable small but degrade when expanded
  • Generic stock photos that do not represent the actual location
  • Images with visible watermarks or Getty/Shutterstock overlays
  • Dark or poorly lit interior shots
  • Portrait-orientation images in the hero banner slot
  • AI-generated images
  • Screenshots from Google Maps or review sites
  • Images with visible people's faces unless it is a supplier-provided photo
Where to Source Images (in priority order)
1

Official Hotel / Supplier Website

Always check the property's official site first. Navigate to their Press / Media Kit section if available — these are the highest quality images, pre-approved for use. For restaurants, check their website gallery. Download the largest version available.

2

Supplier-Provided Images

If the supplier has sent Barbara a folder of images or included them in their confirmation documents, use those first — they are pre-approved and show the exact product the client has booked.

3

Unsplash (unsplash.com)

For destination/city/landmark images, Unsplash is acceptable. Search specifically (e.g., "Fes Morocco medina," not just "Morocco"). Evaluate each result individually — do not take the first result. Choose the sharpest, most evocative image. Note the photographer's name for the credit line.

4

Google Images (last resort only)

If no other source works, use Google Images with "Usage Rights: Creative Commons licenses" filter applied. Verify the license allows commercial use. Never hotlink — always download and re-upload. Flag to Barbara that this image was used and confirm she approves before finalization.

⚠️
Before uploading any image: Open it at full size and zoom to 100%. If there is any softness, grain, or pixelation, do not use it. Barbara reviews the PDF export of itineraries and images must look sharp in print as well as on screen.
Image Check Before Finalizing
  • Every section has exactly one hero image (no section is image-less)
  • Cover hero image is wide-format, destination-specific, and visually striking
  • All images are sharp at 100% zoom — no exceptions
  • Airline images show the correct airline's aircraft
  • Hotel images match the correct property (double-check on hotel website)
  • Restaurant images show the actual restaurant (not a food photo sourced elsewhere)
  • No watermarks visible anywhere
  • Photo credits added for any Unsplash images used
  • No portrait-orientation images used in banner slots
03

QA Review Checklist

The QA review is not optional. Every itinerary must be reviewed top to bottom before being sent to Barbara. Your job is to identify issues, flag them with notes, and let Barbara make the final call. Document every flag — even things that look intentional but seem unusual.
A. Logical Flow & Timing Gaps
  • Arrival logistics are complete. If a flight arrives, is there a transfer waiting? Is the hotel check-in time realistic given the arrival time?
  • No unexplained gaps. If a client has a full free day with no activities listed and no recommendation section, flag it — was this intentional or was something missed?
  • Activity timing makes sense. If a tour starts at 9:00 AM and the hotel is 1.5 hours away, the pickup time must be listed. Verify drive times match the activity start time.
  • Dinner reservations are not booked too early after a long tour. e.g., if an 8-hour tour ends in the afternoon and dinner is at 7:30 PM, there is adequate rest time — this is acceptable. Flag if dinner appears to conflict with an experience end time.
  • Day-long drives account for lunch. Long transfer days (like Fes to Marrakech via Rabat) should note that the guide recommends a restaurant for lunch, or that lunch is included. Flag if a 6-hour drive day has no lunch mention.
  • Departure day logistics are tight-checked. Verify pickup time → drive time → airport arrival → check-in time window → flight departure. Clients should arrive at the airport at least 2 hours before international flights (3 hours for US departures recommended).
  • Connection times are sufficient. If a client has a connection in Athens (like the Greece itinerary: Santorini 8:25 AM → Athens 9:15 AM → Athens 12:35 PM to Chicago), flag if the connection seems under 2 hours for an international flight.
  • Multi-city hotel check-in matches the transfer arrival time. If Villa Makassar in Marrakech has a 2:00 PM check-in but the drive from Fes takes 6+ hours departing at 9:00 AM, the 2:00 PM check-in is close — note this for Barbara.
B. Missing Information
  • All flight record locators are present. Never leave this blank on a final itinerary.
  • All hotel confirmation numbers are entered (final itinerary only — proposal may show "pending").
  • Emergency contact information is complete — local contact names, phone numbers, email, and instructions for how to reach them (WhatsApp group, call, etc.).
  • All restaurant addresses and phone numbers are present. Website URLs where available.
  • Meeting points are specified for every activity (hotel lobby, outside hotel, specific address).
  • Driver/transfer company contact info is present for every transfer.
  • Activity notes are complete — what to wear, what to bring, dietary requirements, difficulty level, etc. If the supplier provided these notes, they must be in the itinerary.
  • Inclusions and exclusions are listed for each major activity (what is/isn't covered in the package).
  • Any costs not included are noted clearly — e.g., "(Cost of dinner is not included)."
  • Dietary requirements noted in the itinerary are actually in the supplier notes (e.g., "The grandparents' vegetarian requests will be accommodated").
  • Client names are spelled correctly throughout, including in the itinerary title and on flights.
  • Passenger seat assignments are included if available.
C. Supplier Terms & Conditions Accuracy
  • T&Cs are present for every supplier that provided them in their confirmation documents.
  • Payment terms are copied verbatim from the supplier's T&Cs — do not paraphrase. If the supplier says "35% deposit non-refundable. 100% 70 days before arrival" that exact language must appear.
  • Cancellation policies are included exactly as stated by the supplier.
  • Inclusions list matches what the supplier confirmed — cross-reference against the confirmation document.
  • Fast Track / VIP service limitations are disclosed — as in Morocco where the T&Cs clarify what Fast Track does and does not include. If the supplier provided this disclaimer, it must be in the itinerary.
  • The "NOTE" about availability changes is present in the supplier T&Cs if the supplier included it.
D. Helpful Information & Client Experience
  • Destination prep content is complete and specific. Does it cover weather, dress code, currency, tipping, shopping, and packing? If any are missing from the supplier's content, flag for Barbara to decide whether to add.
  • Tipping guidance is destination-specific and reflects the actual supplier's recommendations (daily amounts per guide, per chauffeur, currency preference, timing).
  • Guide information section is present for DMC tours — who the guide is, what they handle, any staffing details (e.g., different mountain guides for different activity levels).
  • Special considerations for mixed-ability groups are noted (e.g., different hiking routes for grandparents vs. younger travelers).
  • Attire/what to bring is specified for physical activities (hiking, cruises, desert excursions).
  • Food tour vs. dinner distinction is clear — if a food tasting experience does not replace dinner, that note must appear so clients don't skip their reservation.
  • Axus app download reminder is in the Pre Travel section.
  • Instagram tag reminder is in the Travel Policies section (@BarbaraWeindlingTravels).
  • Restaurant recommendations / free time sections exist for any unstructured half-days or full days.
E. Proofreading
  • Read every description aloud (or carefully) — check for spelling errors, repeated sentences, and awkward phrasing. Note: the Morocco itinerary contained a typo ("grandchildren" → "grabndchildren") and a misspelling ("necessary" → "nececery") — catch these before Barbara sees them.
  • Section titles are consistent and properly formatted (Title Case, with " · " separators for dining reservations).
  • Times are formatted consistently (9:00 AM not 9am or 9:00am).
  • Dates are formatted consistently (Saturday · April 18th).
  • Phone numbers include country codes (e.g., +212 for Morocco, +30 for Greece).
  • No section has duplicate text (e.g., the Morocco itinerary had a repeated paragraph about the chauffeur waiting in the arrival hall — catch and flag these).
  • Colored/highlighted notes (orange text in Morocco) are intentional supplier callouts, not errors — verify they are present and accurate.
F. Proposal-Specific Checks
  • All room rates (nightly and total) are present and match the supplier's quote.
  • All activity/transfer/VIP rates are shown in the Notes field.
  • Currency is correct for each rate (€ for European suppliers, USD for US-origin services).
  • Confirmation references marked as pending are flagged to Barbara to update once confirmed.
  • Payment terms appear in the relevant supplier T&C section.
📬
How to deliver your QA report to Barbara: After completing the review, send a message via Tern with subject "QA Review Complete — [Client Name] [Destination]." List each flag as a numbered item with the section name, the issue, and a suggested fix or question. Do not make changes without Barbara's direction.
