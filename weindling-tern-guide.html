SOP & Workflows

Step-by-Step
Purpose: This SOP outlines workflows and processes for managing travel arrangements using the Tern system, tailored for Barbara's agency (Weindling World Travel).
👤 A. Agency Owner (Barbara)
1. Understand the structure (one-time orientation)

Use the spreadsheet as the master map of everything in Tern:

  • Workflow library — what applies when (Red = inactive by intention; Yellow = manually added/review required)
  • Email templates — what gets sent
  • Forms — what data is collected
2. Decide how to handle leads (Contact vs Trip)
  • Contact-basis Lead Tasks — Use when someone emails/calls but you haven't spoken to them yet. Purpose: capture lead, reply, decide if it becomes a trip.
  • Trip-basis Lead Tasks — Use when you've spoken to someone or are confident it will close. Includes recap email, agreement, and moving to Planning once fee is paid.
  • Either path is acceptable; choose based on confidence level.
3. Planning phase (core daily workflow)

When ready to actively work on a trip:

  1. Change Trip Status → Planning → Add Trips | Planning Workflow

This workflow automatically:

  • Sends Planning Fee Received email (automated)
  • Sends a travel insurance interest reminder (automated)

⚠️ "Tern Access" email remains inactive until you actually send itineraries through Tern.

4. Hotel-Only & Concierge logic

Hotel-Only Trips: Change status → Hotel Only → Add Hotel Only workflow.

Concierge Services (only apply if charging a concierge fee):

  • Add Trip Tag: Concierge
  • In the future this can be auto-applied based on Trip Tag.
5. Trips: Booked Pre/Post Travel

Client approves trip and deposits are paid:

  1. Change Trip Status → Booked → Add Trips | Booked Pre/Post Travel workflow

This applies the Pre/Post Travel workflow, including:

  • Date-based tasks tied to trip start/end
  • A required early task to: add trip-specific tasks, personalize emails, activate selected automations
  • You can still add one-off tasks (e.g., museum tickets not released yet)
6. Automations: what stays active vs inactive

Active: 45-day check-in (includes emergency contact + passport info form)

Inactive by design (must be personalized before activating):

  • Bon Voyage
  • Welcome Home
  • Passport Control / Global Entry (only if applicable to client)
7. Cruise workflows (Travel Biz Boss)

Two cruise workflows in the library:

  • Mass Market Cruise — Set to Auto-apply with Trip Tag Prem/Lux river
  • Premium / Luxury Cruise

Best practice: Turn off email notifications to yourself (too noisy). Automations are cruise-line agnostic, triggered by Trip Start Date.

8. Cruise email templates (important rule)
Do not copy cruise templates into your library unless you plan to edit them — if you copy them, you'll miss future updates from Travel Biz Boss.

Templates include: deposit & final payment emails per cruise line, links to apps/FAQs/packages/T&Cs, packing lists and destination guidance.

9. Email template housekeeping
  • Review folders: General client emails & Hotel supplier communications
  • These live in Tern so a VA can access them — no shared drives or Outlook folders needed
  • One highlighted template needs a decision: review revised copy and decide whether to update
10. Forms

Forms are already organized in the library. Folder location indicates usage. No action required unless you want to rename or consolidate.

🧑‍💼 B. Admin / Virtual Assistant (VA)
1. Where the VA works

VA operates inside Tern, not email inboxes. Uses task lists applied to trips and shared email templates (cruise templates are shared, not copied).

2. Sending cruise emails
  1. Barbara confirms payment action is needed
  2. VA sends appropriate template via Tern
  3. No editing required unless instructed
3. Supplier communications

VA can send supplier emails using Tern templates. Emails can be scheduled and associated with the correct trip. This keeps a clean record in the Trip > Emails section.

4. Task management

VA completes tasks assigned via workflows: supplier follow-ups, document checks, post-trip follow-up tasks. Does not activate automations unless instructed.

5. What the VA should NOT do
  • Activate inactive automations (until Barbara indicates otherwise)
  • Edit master workflows
  • Copy Travel Biz Boss templates into the library
  • Change trip statuses without approval

Payment Reminder Workflow

Setup Guide
Current limitation: Credit cards are not captured in Tern, so payment reminders are based on Trip Start Date, not payment due date. Use the payment calculator to determine the correct number of days.
📋 Step-by-Step Setup

  1. Trigger the workflow: Open the trip record. Confirm Trip Start Date and final payment due date are entered. Change Trip Status to Booked.

  2. Confirm tasks were created: After status changes to Booked, verify the following appear in the task list.

  3. Activate the workflow: Open task "Activate payment reminders to send to client." Confirm email notification is OFF (internal task only). Mark the task Complete.

  4. Adjust timing per trip: Edit the 62-day or 32-day reminder as needed (e.g., change 62 → 48 days based on actual payment due date).

  5. Multiple suppliers: If trip has multiple suppliers with varying due dates, add the Payment Reminder automation template multiple times — one per payment.

📊 Default Payment Reminder Tasks
Task NameDue Date LogicDefault Status
Activate payment reminders to send to client 2 days after task creation Active
Payment Reminder (62 days before trip start) Trip Start Date – 62 days Inactive
Payment Reminder (32 days before trip start) Trip Start Date – 32 days Inactive — Manual activate only if needed
🧮 Payment Calculator — How to Use

  1. Open the Payment Calculator worksheet in the spreadsheet.

  2. Enter the trip start date and the payment due date.

  3. The calculator subtracts the trip start date from the payment due date and adds a 10-day buffer.

  4. Use the result (number of days) when setting up your payment reminder trigger in Tern.

Example: Trip Start = June 15, 2026. Final payment due = February 15, 2026 (120 days before trip start). Enter these dates → calculator returns the correct trigger day offset including the 10-day buffer.
